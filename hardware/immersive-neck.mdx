The immersive neck is a 2-DoF actuated mount for a ZED Mini that follows the operator's head. This guide covers building and calibrating one.

<Frame caption="The assembled immersive neck">
  <img src="/images/immersive-neck.png" alt="Assembled immersive neck: ZED Mini on a 3D-printed 2-DoF servo mount" />
</Frame>

## Requirements

- ZED Mini
- 2× Feetech STS3215-C018 servos
- Motor control board
- USB-C cable
- 12 V DC power supply
- Windows PC (for motor ID configuration)
- 3D printer + PLA

## Build and setup

<Steps>
  <Step title="Print the parts">
    Download and print the STL files from [Google Drive](https://drive.google.com/drive/folders/1LuZOyft3a99CJ9f7cuppR4ZS39v8eokr?usp=sharing).
  </Step>

  <Step title="Configure motor IDs (Feetech)">
    Install the [Feetech motor software](https://www.feetechrc.com/software.html) (Windows only). Open it and set:
    - Bottom (yaw) motor → **ID = 2**
    - Top (pitch) motor → **ID = 1**
  </Step>

  <Step title="Assemble the neck">
    - Insert the motors into the printed parts.
    - Wire the motors to the motor control board.
    - Mount the neck on a tripod stand or clamp it down securely.
  </Step>

  <Step title="Calibrate">
    Download <a href="/scripts/calibrate.py" download>`calibrate.py`</a> and run it. It needs the Feetech `scservo_sdk` Python package (`pip install feetech-servo-sdk`).

    ```shell
    python3 calibrate.py --port <port> --output <path>/neck_calibration.json
    ```

    <Warning>
      If you hit serial-port permission issues, try `sudo chmod 777 -R /dev/`.
    </Warning>
  </Step>

  <Step title="Test the limits">
    Verify the calibration with <a href="/scripts/test_limits.py" download>`test_limits.py`</a>:

    ```shell
    python3 test_limits.py --port <port> --calibration-file <path to calibration>
    ```
  </Step>

  <Step title="Install the ZED">
    <Warning>
      Run `test_limits.py` **before** installing the ZED. Incorrect calibration may damage the cameras and motors.
    </Warning>

    Mount the ZED camera in the neck.
  </Step>
</Steps>

<Check>
  Once the hardware is working, reach out to the Avea team for the required config changes (see [Example configs](/configuration/examples) for immersive-neck setups).
</Check>
