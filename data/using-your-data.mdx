If dataset recording is enabled in your `robot.yaml`, Sentinel records teleoperation sessions to MCAP files for downstream training and analysis. Each session contains (by default):

- **XR input** — controller tracking, buttons, joysticks
- **Robot state** — joint positions, velocities, gripper states
- **Robot commands** — joint commands, gripper commands
- **Camera streams** — encoded video from all configured cameras
- **System events** — state transitions, errors, telemetry

<Note>
  Which topics are recorded is fully configurable through your `robot.yaml`. See [`dataset_recording`](/configuration/reference#dataset-recording).
</Note>

## Dataset directory structure

```text
datasets/
└── session_YYYYMMDD_HHMMSS/          # Session directory (timestamp-based)
    ├── episode_001/                   # Individual episode
    │   ├── episode_001_0.mcap         # MCAP bag file (split at 1GB; configurable)
    │   ├── episode_001_1.mcap         # Additional splits if needed
    │   └── sentinel_metadata.json     # Episode metadata
    ├── episode_002/
    │   ├── episode_002_0.mcap
    │   └── sentinel_metadata.json
    └── episode_003/
        ├── episode_003_0.mcap
        └── sentinel_metadata.json
```

**Naming convention**
- Session ID: `session_YYYYMMDD_HHMMSS` (e.g. `session_20260418_143052`)
- Episode ID: `episode_NNN` (zero-padded, e.g. `episode_001`)
- MCAP files split at 1 GB max, numbered sequentially

## Message definitions

All data is stored as serialized ROS 2 messages. Joint commands and joint states use standard `JointTrajectory` and `JointState`. The remaining messages use custom types, defined below.

### XRInput

```text
std_msgs/Header header

# All tracking data (controllers, HMD, skeleton, hands)
XRTransform[] transforms

# Controller buttons
ButtonStates buttons

# Joystick inputs
JoystickInput joystick
```

**XRTransform**
```text
string name                     # "controller_left", "controller_right", "hmd", etc.
geometry_msgs/Transform transform
float32 confidence              # 0.0 to 1.0
bool is_tracked
```

**ButtonStates**
```text
bool a_button
bool b_button
bool x_button
bool y_button
bool menu_button
float32 left_trigger            # 0.0 (released) to 1.0 (pressed)
float32 right_trigger
float32 left_grip
float32 right_grip
```

**JoystickInput**
```text
float32 left_x                  # -1.0 (left) to +1.0 (right)
float32 left_y                  # -1.0 (down) to +1.0 (up)
float32 right_x
float32 right_y
```

### SystemState

**Topic:** `/{namespace}/sentinel/system/state/current`
**Type:** `sentinel_msgs/msg/SystemState`

```text
uint8 state                              # State ID (see constants)
string state_name                        # "DISARMED", "ARMED", "TELEOP_ACTIVE", etc.
builtin_interfaces/Time entered_at       # When the state was entered
```

**Constants**
```text
uint8 DISARMED = 0
uint8 ARMING = 1
uint8 ARMED = 2
uint8 CALIBRATING = 3
uint8 HOMING = 4
uint8 TELEOP_ACTIVE = 5
uint8 E_STOP = 6
```

### JointState

**Topic:** `/{namespace}/sentinel/robot/state/joint_states/measured`
**Type:** `sensor_msgs/msg/JointState`

```text
std_msgs/Header header
string[] name                   # Joint names
float64[] position              # radians
float64[] velocity              # rad/s
float64[] effort                # torques/forces (optional)
```

### GripperState

**Topic:** `/{namespace}/sentinel/robot/state/grippers`
**Type:** `sentinel_msgs/msg/GripperState`

```text
std_msgs/Header header

# Left gripper (normalized 0.0-100.0)
float64 left_opening
float64 left_speed
float64 left_force

# Right gripper (normalized 0.0-100.0)
float64 right_opening
float64 right_speed
float64 right_force

string result                   # "success", "fail_motor", etc.
```

### GripperCommand

**Topic:** `/{namespace}/sentinel/control/commands/gripper`
**Type:** `sentinel_msgs/msg/GripperCommand`

```text
std_msgs/Header header

string grasp_type               # "open_close", "pinch", "power", "tripod"
float64 command                 # 0.0 (closed) to 100.0 (open)
string hand                     # "left", "right", "both"

# Optional parameters
float64 speed                   # 0-100
float64 force                   # 0-100
float64 max_effort              # Legacy parameter
```

### EncodedImage

**Topic:** `/sentinel/vision/output/{camera_name}/encoded`
**Type:** `sentinel_msgs/msg/EncodedImage`

```text
std_msgs/Header header

# Codec configuration
string encoding                 # "h264", "h265", "jpeg"
string profile                  # "baseline", "main", "high"
string level                    # "3.1", "4.0", etc.
uint32 width
uint32 height
uint32 bitrate
uint32 framerate

# Color information
string color_space              # "bt601", "bt709", "bt2020"
string color_range              # "limited", "full"

# GOP structure
uint32 gop_size                 # Keyframe interval
uint32 bframes

# Per-frame metadata
uint8 frame_type                # IDR=1, I=2, P=3, B=4
uint64 sequence_number
uint64 pts_ns                   # Presentation timestamp (ns)
uint64 dts_ns                   # Decode timestamp (ns)
uint32 frame_duration_ns

# Encoding metrics
uint64 encode_timestamp_ns
uint32 encode_duration_us
uint32 frame_size_bytes

# RTP metadata
uint32 rtp_timestamp
uint32 payload_type
uint32 ssrc

# Codec-specific data (SPS/PPS for H.264)
uint8[] codec_config_data

# Encoded frame data
uint8[] data
```

### Decoding EncodedImage

A minimal example using PyAV (FFmpeg bindings):

```python
#!/usr/bin/env python3
"""Decode sentinel_msgs/EncodedImage (H.264/H.265) using PyAV.

    pip install av
"""

import rclpy
from rclpy.node import Node
from rclpy.qos import QoSProfile, ReliabilityPolicy, HistoryPolicy
from sentinel_msgs.msg import EncodedImage
import av


class DecodeExample(Node):
    def __init__(self, topic):
        super().__init__('decode_example')
        self.codec_context = None
        self.frames_decoded = 0

        # BEST_EFFORT QoS is typical for video streaming
        qos = QoSProfile(
            reliability=ReliabilityPolicy.BEST_EFFORT,
            history=HistoryPolicy.KEEP_LAST,
            depth=5,
        )
        self.sub = self.create_subscription(EncodedImage, topic, self.callback, qos)
        self.get_logger().info(f"Listening on: {topic}")

    def callback(self, msg):
        if self.codec_context is None:
            codec_name = 'h264' if msg.encoding == 'h264' else 'hevc'
            self.codec_context = av.CodecContext.create(codec_name, 'r')

        try:
            packet = av.Packet(bytes(msg.data))
            packet.pts = msg.pts_ns
            packet.dts = msg.dts_ns if msg.dts_ns != 0 else msg.pts_ns

            for frame in self.codec_context.decode(packet):
                rgb_array = frame.to_ndarray(format='rgb24')  # [height, width, 3]
                self.frames_decoded += 1
                # Use the decoded frame here (display, save, process)
        except Exception as e:
            self.get_logger().error(f"Decode error: {e}")


def main():
    import argparse, sys
    parser = argparse.ArgumentParser(description='Decode EncodedImage example')
    parser.add_argument('--topic', default='/sentinel/vision/output/wrist/encoded')
    args = parser.parse_args([a for a in sys.argv[1:] if not a.startswith('--ros-args')])

    rclpy.init()
    node = DecodeExample(args.topic)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()


if __name__ == '__main__':
    main()
```

## Episode metadata — `sentinel_metadata.json`

Written per-episode as a quick-to-parse summary of what was recorded.

- `annotations` — user/system labels, e.g. `{ "outcome": "success" }`
- `bag_info` — `bag_size_bytes` and `message_count`
- `duration_seconds` — episode duration (float)
- `episode_id` / `episode_number` — e.g. `episode_001` / `1`
- `session_id` — groups episodes under one teleop session
- `started_at` / `stopped_at` — UTC ISO-8601 timestamps
- `system_states` — list of state transitions during the episode (`[]` if none)
- `topics` — the ROS topics recorded
