The Sentinel runtime ships as a container you run on a computer near your robot. In most cases we provide a tuned `robot.yaml` config for your hardware — you add it, run the launcher, and drive.

<Note>
  **Latest version:** `0.11.0`. Versions below `0.7.0` are no longer supported and cannot be used. `0.7.0` introduced major breaking changes — configs made for earlier versions must be replaced. Contact us to update your config.
</Note>

## Overview

Sentinel is a modular, container-based runtime for robot teleoperation. It supports multiple deployment configurations — on-robot and companion-computer setups — and is designed to be flexible across hardware targets.

## Requirements

**Platform**
- Ubuntu 22.04 or 24.04 (x86)
- NVIDIA GPU with CUDA 13+ *or* a Jetson device running L4T R36+ (JetPack 6.2.x)

**Dependencies**
- [Docker Engine v27+](https://docs.docker.com/engine/install/ubuntu/)
- [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html)
- [AWS CLI](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html) — we recommend installing via snap
- 10 GB of free space on the root filesystem
- A valid `robot.yaml` config file

## Setup

<Steps>
  <Step title="Create a local Sentinel folder">
    On your teleop companion computer, create a clean folder to hold your config file (`robot.yaml`) and the launcher script (`run_sentinel.sh`):

    ```shell
    mkdir -p ~/sentinel
    cd ~/sentinel
    ```
  </Step>

  <Step title="Add your robot config file">
    Save the config we provided as `robot.yaml` in this folder. The provided config has a placeholder license key you'll replace below.

    ```shell
    ls -la ~/sentinel
    # should include: robot.yaml
    ```
  </Step>

  <Step title="Save the launcher script">
    Create `run_sentinel.sh` in `~/sentinel` and paste the script below:

    ```bash
    #!/bin/bash
    set -e

    SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
    REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
    [[ -f "$SCRIPT_DIR/.env" ]] && source "$SCRIPT_DIR/.env"

    CONFIG_FILE="$1"
    VERSION="${2:-latest}"
    NECK_CALIBRATION="${3:-}"
    CONTAINER_NAME="${4:-sentinel-runtime}"

    if [[ -z "$CONFIG_FILE" ]]; then
        cat >&2 <<EOF
    Usage: $0 <config_file> [version] [neck_calibration_file] [container_name]
      $0 /abs/path/yam_dual.yaml 0.7.0-amd64
      $0 /abs/path/cfg.yaml      0.7.0-jp6.2 /abs/path/neck_calibration.json
      $0 /abs/path/cfg.yaml      0.7.0-jp6.2 /abs/path/neck_calibration.json my-name
    EOF
        exit 1
    fi
    [[ -f "$CONFIG_FILE" ]] || { echo "ERROR: config not found: $CONFIG_FILE" >&2; exit 1; }
    [[ -z "$NECK_CALIBRATION" || -f "$NECK_CALIBRATION" ]] \
        || { echo "ERROR: neck calibration not found: $NECK_CALIBRATION" >&2; exit 1; }

    AUTH_URL="https://api-prod.avearobotics.com"
    LICENSE_KEY="$(python3 - "$CONFIG_FILE" <<'PY'
    import sys, yaml
    sys_ = (yaml.safe_load(open(sys.argv[1])) or {}).get('system') or {}
    print(sys_.get('license_key') or '')
    PY
    )"
    [[ -n "$LICENSE_KEY" ]] \
        || { echo "ERROR: system.license_key not found in $CONFIG_FILE" >&2; exit 1; }

    ECR_RESP="$(curl -fsS -X POST "$AUTH_URL/auth/robot/ecr-credentials" \
        -H 'Content-Type: application/json' \
        -d "{\"license_key\":\"$LICENSE_KEY\"}")"
    [[ -n "$ECR_RESP" ]] || { echo "ERROR: empty ECR response from $AUTH_URL" >&2; exit 1; }

    mapfile -t CREDS < <(printf '%s' "$ECR_RESP" | python3 -c \
        'import json,sys;r=json.load(sys.stdin);print(r["registry"]);print(r["password"])')
    REGISTRY="${CREDS[0]}"
    printf '%s' "${CREDS[1]}" \
        | docker login --username AWS --password-stdin "$REGISTRY" >/dev/null
    unset CREDS ECR_RESP

    IMAGE="$REGISTRY/sentinel-runtime:${VERSION}"

    ARCH=$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/')
    if [[ "$ARCH" == "arm64" && -f /etc/nv_tegra_release ]]; then
        PLATFORM=jetson
    elif [[ "$ARCH" == "amd64" ]] && command -v nvidia-smi >/dev/null && nvidia-smi -L >/dev/null 2>&1; then
        PLATFORM=cuda
    else
        PLATFORM=cpu
    fi

    echo "sentinel-runtime $VERSION  $PLATFORM/$ARCH  $IMAGE  ($CONTAINER_NAME)"

    cd "$REPO_ROOT"
    mkdir -p datasets

    ARGS=(
        -it --rm --name "$CONTAINER_NAME"
        --privileged --network host --ipc host
        -v /dev:/dev -v /tmp:/tmp
        -v "$REPO_ROOT/datasets:/datasets"
        -v "$(realpath "$CONFIG_FILE"):/config/robot.yaml:ro"
        -v zed-models:/usr/local/zed/resources
        --group-add video --group-add plugdev
        -e RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
        -e ENABLE_RECORDING=true
    )

    case "$PLATFORM" in
        cuda)
            ARGS+=(--gpus all -e NVIDIA_VISIBLE_DEVICES=all -e NVIDIA_DRIVER_CAPABILITIES=all) ;;
        jetson)
            ARGS+=(--runtime nvidia --gpus all
                   -e NVIDIA_VISIBLE_DEVICES=all -e NVIDIA_DRIVER_CAPABILITIES=all)
            # ZED X / GMSL: Argus ISP settings + host zed/i2c/render GIDs.
            # /tmp is already bound, so argus_socket + zed_x_daemon sockets come along.
            [[ -d /var/nvidia/nvcam/settings ]] \
                && ARGS+=(-v /var/nvidia/nvcam/settings:/var/nvidia/nvcam/settings:ro)
            for grp in zed i2c render; do
                gid=$(getent group "$grp" 2>/dev/null | cut -d: -f3 || true)
                [[ -n "$gid" ]] && ARGS+=(--group-add "$gid")
            done
            ;;
    esac

    [[ -n "$NECK_CALIBRATION" ]] \
        && ARGS+=(-v "$(realpath "$NECK_CALIBRATION"):/config/neck_calibration.json:ro")

    exec docker run "${ARGS[@]}" "$IMAGE"
    ```

    Then make it executable:

    ```shell
    chmod +x run_sentinel.sh
    ```

    Your folder should now contain `robot.yaml` and `run_sentinel.sh`.
  </Step>

  <Step title="Get a license key">
    Your `robot.yaml` ships with a placeholder key:

    ```yaml
    system:
      license_key: "PLACEHOLDER_KEY"
    ```

    Generate a live key in the [Avea Robotics dashboard](https://dashboard.avearobotics.com/dashboard/robots) → **Add Robot**:

    <Frame>
      <img src="/images/dashboard-add-robot.png" alt="The Robots page of the Avea dashboard with the Add Robot button highlighted" />
    </Frame>

    Then replace the placeholder:

    ```yaml
    system:
      license_key: "lk_c41d041c-..."
      enable_dataset_recording: true
    ```
  </Step>

  <Step title="Run Sentinel">
    Pick a version and platform tag and launch:

    ```shell
    cd ~/sentinel
    ./run_sentinel.sh ./robot.yaml 0.11.0-amd64-cu13
    ```

    Optionally include a neck calibration file (absolute path required):

    ```shell
    ./run_sentinel.sh ./robot.yaml 0.11.0-amd64-cu13 /absolute/path/to/neck_calibration.json
    ```

    Once it's running, open [Sentinel XR Teleoperation](http://app.avearobotics.com) on your Meta Quest 3 / 3S to control your robot.
  </Step>
</Steps>

## Supported platform tags

- `amd64-cu13` — x86 CPU + NVIDIA discrete GPU (CUDA 13+)
- `jp6.2` — Jetson Orin running JetPack 6.2.1 or 6.2.2
- `amd64` — x86 CPU only
- `arm64` — arm64 computer only

<Warning>
  **Always prune old image versions before pulling a new one.** Sentinel is safety-critical — running mismatched versions can produce undefined behavior.
</Warning>
