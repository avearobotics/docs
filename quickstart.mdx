This guide takes you from a robot to a safe VR teleoperation session. You will connect the robot, run the Sentinel runtime, verify its state, and start operating.

## Choose your setup path

<CardGroup cols={2}>
  <Card title="Supported robot" icon="bolt" href="/hardware/supported">
    Use a tested integration and a ready-made `robot.yaml`. This is the shortest path.
  </Card>
  <Card title="ROS 2 robot" icon="diagram-project" href="/integration/overview">
    Map your existing command, joint-state, and camera topics to Sentinel.
  </Card>
</CardGroup>

If your hardware is on the supported list, you do not need to build an adapter. If it is not, most ROS 2 robots connect through standard topics without running Sentinel code inside the robot controller.

## Before you start

- A supported computer running Ubuntu 22.04 or 24.04
- Docker and the required NVIDIA runtime for your platform
- A Meta Quest 3 or 3S
- A clear robot workspace and access to the robot's emergency stop
- A Sentinel account, registered robot, and license key
- A ready-made or completed `robot.yaml`

See [Installation](/installation) for the current compute, storage, and software requirements.

## From setup to operation

<Steps>
  <Step title="Connect your robot">
    For supported hardware, download the matching [example config](/configuration/examples). For another ROS 2 robot, complete the [ROS 2 connection guide](/integration/overview) and verify your command, state, and camera topics with standard ROS 2 tools.
  </Step>

  <Step title="Register the robot">
    Add the robot in the [Sentinel dashboard](https://dashboard.avearobotics.com/dashboard/robots), generate its license key, and place the key in `system.license_key` in `robot.yaml`.
  </Step>

  <Step title="Install and launch Sentinel">
    Follow [Installation](/installation) to install the prerequisites and start the runtime with your config. Keep the terminal visible during the first run so you can see connection and safety errors.
  </Step>

  <Step title="Verify the robot is ready">
    Confirm the runtime sees robot state and the expected cameras. The robot should remain disarmed and still. Fix missing state, incorrect joint names, or camera errors before continuing.
  </Step>

  <Step title="Complete your first VR session">
    Put on the headset, choose the robot, arm it, and start teleoperation only after the workspace is clear. Follow [Your first session](/operate/first-session) for the control and safety sequence.
  </Step>

  <Step title="Record one episode">
    Start a short recording, perform one task, then stop the recording. Sentinel stores the episode locally as MCAP with robot state, commands, camera streams, and system events. See [Collect data](/data/overview).
  </Step>
</Steps>

## You are done when

- Robot state and cameras appear without errors.
- The robot stays still until you deliberately arm and begin teleoperation.
- Releasing the controller grip stops teleoperation and holds the robot safely.
- A recorded episode appears in the local dataset directory.

<Warning>
  Treat first motion as a bring-up procedure. Keep the workspace clear, stay within reach of the physical emergency stop, begin at conservative speed and limits, and confirm the robot's hold behavior before attempting a task.
</Warning>
